Model Fitting software

This sections only detail GUI part

Requested features:

  • Add document modified flag into the whole code . That is really important because work can be lost after load or new menu request!
  • Propose on parameters view that group every parameter of one group
  • Make right click on parameters to share them without selection need
  • Request to the server the synthetic oifits file that according given model
  • Finish result plots
    • radial plots
    • image
    • sniffer plot
  • Use checkboxes to file selection instead of the simple list widget
  • Permit user to select installed programs as fits viewer
  • Add one format version of all xml settings files (Future revision will be able to check format validity and propose converter tools)
  • Improve velocity of result display and show running actions( user sometimes click several times on buttons because nothing seems running...)

Implemented features:

V1.0.6.beta

  • Show most of informations about fitting process into result panel
  • Result panel starts to plot yoga results
    • baselines
    • uvcoverage
    • model image
    • model uv map
  • Do not return error not enough data point anymore due to blank names into oitarget names
  • A simple image viewer has been implemented with 3 selectable color models

V1.0.5.beta

  • Handle better oitargets name even if new fits library places some spaces into oitarget names
  • Use advanced menu to group interop menu and logging utility
  • Show covariance matrix into result panel

V1.0.4

  • Fitting results are taken into account for parameters values
  • Models can share parameters
  • Fits files with zero length column supported
  • Fits viewer do not close anymore main application while quitting fits viewer
  • UV Coverage plot and interferometer sketch are automated at one basic level

V1.0.3:

  • Select by default oifiles associated to the added target
  • Display more info on observed data files
  • Make possible to use xml model without embedded b64 files. If the file is not found during opening, a dialog asks to locate it.
  • Quit action checks that every document have been saved

V1.0.2:

  • Display the release panel
  • Saving file does not use openDialog anymore but saveDialog instead

V1.0.1:

  • Fits file viewers can show selected tables with topcat defaut viewer
  • Embedded data extracted one time for every actions(viewing,saving...)

V1.0:

  • FilePanel permits to save embedded file
  • Delete button used to supress target refreshes the left tree
  • Preference window close button does not quit the application anylonger
  • Double clicks into list select view on clicked element
  • Settings splits sub components grouping FilesPanel and TargetsPanel
  • Report error loading bad format files (oifiles and settings ones)
